# 订单处理平台 — Docker 部署指南（Windows）

## 一、项目文件结构

部署前，确保你的项目文件夹结构如下：

```
订单处理平台/
├── app.py
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
├── .dockerignore
└── templates/
    └── index.html
```

---

## 二、安装 Docker Desktop

1. 打开浏览器，访问 https://www.docker.com/products/docker-desktop/
2. 点击 **Download for Windows** 下载安装包
3. 双击安装包，一路 Next 完成安装
4. 安装完成后**重启电脑**
5. 重启后桌面右下角任务栏会出现 Docker 鲸鱼图标，等它变成**绿色**（Running）

> ⚠️ 如果提示需要开启 WSL2，按照提示点击链接安装即可，安装后再次重启。

---

## 三、启动项目（只需做一次）

打开 **PowerShell** 或 **命令提示符（CMD）**：

```powershell
# 1. 进入项目文件夹（路径改成你自己的）
cd C:\Users\你的用户名\Desktop\订单处理平台

# 2. 构建并启动容器（首次需要下载镜像，约 200MB，需要等几分钟）
docker compose up --build -d

# 3. 查看是否启动成功
docker compose ps
```

看到 `Status: running` 就说明启动成功了。

---

## 四、使用平台

打开浏览器，访问：

```
http://localhost:5000
```

---

## 五、日常操作命令

```powershell
# 启动（已经 build 过后，日常启动用这个）
docker compose up -d

# 停止
docker compose down

# 查看运行日志（出错时用）
docker compose logs -f

# 重新构建（修改了代码后需要重新构建）
docker compose up --build -d
```

## 六、更新代码

```powershell
cd C:\Users\你的用户名\Desktop\订单处理平台

```

## 七、常见问题

**Q：浏览器打开 localhost:5000 显示"无法访问此网站"**
- 检查 Docker Desktop 是否在运行（任务栏鲸鱼图标是绿色）
- 执行 `docker compose ps` 确认容器状态是 `running`
- 执行 `docker compose logs` 查看错误信息

**Q：端口 5000 被占用**
- 打开 `docker-compose.yml`，把 `"5000:5000"` 改成 `"8080:5000"`
- 然后访问 `http://localhost:8080`

**Q：上传文件很慢**
- 这是正常现象，大文件上传需要时间
- 默认支持最大 500MB 文件

**Q：容器重启后数据还在吗**
- 处理中的临时文件在 `workspace/` 文件夹，已映射到本机
- 下载完成后会自动清除，正常情况下无需手动管理
